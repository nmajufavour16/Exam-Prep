Operating System Basics

Secondary Definition (Hardware)

Hardware View An OS is an Input Unit that translates or converts all the tools to work with into the Computer — RAM/ROM. RAM = Random Access Memory. ROM = Read Only Memory.

Topic 02

Deadlock

Definition (By Definition) Deadlock can be defined as a set of processes where each process in the set is waiting for an event that can be caused by another process in that set. When something is hindering a process from doing its thing — it is deadlocked. When you start a process and it didn't finish, that process is also deadlocked.
Alternative Definition (5/02/24) A Deadlock is therefore a situation where a group of processes are permanently blocked as a result of each process having acquired a subset of the resources needed for its completion, and waiting for release of the remaining resources held by others in the same group — making it impossible for any process to prevail.

5 Conditions for Deadlock Occurrence

  • Mutual Exclusion — When there is an agreement between two processes (only one can use resource at a time)
  • Hold and Wait — A process holds a resource while waiting for another
  • No Preemption — Resources cannot be forcibly taken from a process
  • Circular Waiting — Each process waits for a resource held by the next in a circle
  • Irreversible — Two processes cannot agree; deadlock has occurred

4 Methods of Dealing with Deadlock

1. Prevention

The OS needs to use some explicit action to deal with possible deadlock before it occurs. Eliminate one of the 5 conditions.

2. Avoidance

Use algorithms (like Banker's Algorithm) to ensure the system never enters an unsafe state.

3. Detection

Allow deadlock to occur, detect it using resource allocation graphs or tables, then resolve.

4. Recovery

After detection, recover by terminating processes or preempting resources.

Banker's Algorithm

Concept When there is a completion, you can say a state is Safe or Unsafe. The Banker's Algorithm helps determine if granting a resource request will leave the system in a safe state.

Safe vs Unsafe State

Safe State ✓

When one user/process can borrow and get to completion, you can say the state is safe. When you borrow and get to the completion, you give it back to the pool so others can also use it.

Unsafe State ✗

Unsafe state is when none of the competitors can get to the end. The unsafe state does NOT necessarily imply the eventual existence of deadlock — but it could lead there.

Worked Example — State 2

Key Rules

  • We want to migrate from Safe State to Unsafe State back to Safe State
  • When a process borrows and completes, it returns resources to the pool
  • Need = Maximum Need − Current Loan
  • If Need ≤ Available → process can proceed (safe)

Page Memory Allocation (27/02/2025)

Definition Before executing a program, the memory manager does the following: determines the number of pages in a program, locates enough empty page frames in main memory, loads all the program pages into starter pages.

Key Page Memory Concepts

Q01 — DefinitionDefine an Operating System using two definitions.
Definition 1 (Program-based): An Operating System is an elaborate set of programs developed by a programmer to control all the operations that go on within the computer, to such an extent that it does not require any human intervention. It has three phases: Input, Process, and Output.

Definition 2 (Hardware-based): An Operating System is an Input Unit that translates or converts all the tools to work with into the Computer — RAM (Random Access Memory) and ROM (Read Only Memory). When you boot a system, the ROM calls the OS, tools, editors, compilers, assemblers, and interpreters.
Q02 — DeadlockWhat is a Deadlock? State the five conditions necessary for a deadlock to occur.
Definition: A Deadlock is a situation where a group of processes are permanently blocked because each process has acquired a subset of the resources it needs and is waiting for the remaining resources held by others in the same group, making it impossible for any process to proceed.

Five Conditions (ALL must be present):
  • Mutual Exclusion — Only one process can use a resource at a time
  • Hold and Wait — A process holds resources while waiting for others
  • No Preemption — Resources cannot be forcibly taken away
  • Circular Wait — Each process waits for a resource held by the next in a circular chain
  • Irreversibility — The deadlock cannot resolve itself
Q03 — DeadlockWhat are the four methods of dealing with deadlock?
  • Prevention — Eliminate one of the five necessary conditions before it can occur. The OS uses explicit actions to preclude deadlock.
  • Avoidance — Use algorithms (like Banker's Algorithm) to ensure the system never enters an unsafe state.
  • Detection — Allow deadlock to occur, then detect it using resource allocation tables or graphs.
  • Recovery — After detection, resolve by terminating processes or preempting/reallocating resources.
Q04 — Banker's AlgorithmExplain the Banker's Algorithm and distinguish between Safe and Unsafe states.
Banker's Algorithm: A deadlock avoidance algorithm used by the OS to determine whether granting a resource request will leave the system in a safe state. It simulates allocation and checks if all processes can still complete.

Safe State: When at least one process can borrow resources, execute to completion, and return resources to the pool — allowing others to proceed. The system can satisfy all future resource requests.

Unsafe State: When none of the processes can get enough resources to complete. Note: An unsafe state does NOT necessarily mean deadlock has occurred — but it could lead to one.

Formula: Need = Maximum Need − Current Loan. If Need ≤ Available resources → process can proceed safely.
Q05 — MemoryWhat is Memory Management? List and briefly explain the types.
Memory Management is basically subdivided into Old and New techniques:

Early (Old) Memory Management:
  • Single User Configuration (SVC) — One user at a time; associated with batch OS like MS-DOS
  • Partition Memory Management — Memory is divided into fixed partitions
  • Dynamic Memory Management — Partitions created as needed
  • Relocatable Memory Management — Programs can be moved in memory
Recent (New) Memory Management:
  • Paged Memory Allocation — Memory divided into equal-sized pages
  • Demand Paged Memory Allocation — Pages loaded only when needed
  • Segmented Memory Allocation — Memory divided by logical segments
  • Segmented/Demand Paged Allocation — Combination of segmentation and demand paging
Q06 — MemoryExplain Page Memory Allocation with an example.
Page Memory Allocation: Before executing a program, the memory manager: (1) determines the number of pages in the program, (2) locates enough empty page frames in main memory, and (3) loads all the program pages into starter pages.

Pages are usually mapped in bytes (8 bits = 1 byte). Pages can be non-contiguous — they can be at different physical locations but appear contiguous to the program.

Example: A job with 1100 lines, with page size = 350 lines:
  • First page: 100 lines
  • Second page: 100 lines
  • Third page: 100 lines
  • Remaining 50 lines → Wasted Space (internal fragmentation)

SVC: Associated with a standalone computer running a batch operating system (e.g. IBM OS/370, FORTRAN, MS-DOS).

Characteristics:

Q08 — InterruptsWhat is Interrupt Processing? Name and explain the four types.
Interrupt Processing: "Another word for Interrupt is After" — it refers to a signal that causes the CPU to suspend its current task and handle another event.

Four Types:
  • Program Interrupt — Caused by a software error, e.g. dividing any number by zero
  • Timer Interrupt — Triggered by a timer/clock (→ CMOS Battery)
  • Input/Output Interrupt — Caused by I/O devices completing or needing attention
  • Hardware Failure — Caused by a hardware fault or failure

  • Memory Manager — Controls and tracks all memory allocation and de-allocation; ensures programs get the memory they need
  • Processor Manager — Manages CPU time; schedules which process gets the CPU and for how long (scheduling)

Q11 — MultiDistinguish between Multi-programming, Multi-processing, and Batch Processing.
Multi-programming: The process where two or more programs are loaded into memory at the same time. When one program waits (e.g. for I/O), the CPU switches to another. Maximizes CPU utilization.

Multi-processing: Two or more CPUs work together. The same job (task) can be jumping from one CPU to another. Provides true parallel execution.

Batch Processing: Batch means group. Jobs come in as a group (batch) and are processed without user interaction. The system (called a Batch System) processes them sequentially without interruption. Different from multi-programming in that the user submits jobs and leaves — no real-time interaction.
Q12 — DiningDescribe the Dining Philosophers Problem and explain its relevance to OS concepts.
The Dining Philosophers Problem: A classic synchronization problem in OS. Imagine 5 philosophers seated at a round table. Each philosopher alternates between thinking and eating. Between each pair of philosophers is one fork — so 5 forks total. To eat, a philosopher needs TWO forks (left and right).

The Problem: If all philosophers pick up their left fork simultaneously, each waits for the right fork — held by their neighbour — creating a deadlock. No one can eat.

Relevance: Illustrates deadlock (circular wait) and resource allocation in concurrent systems. Solutions include: letting only 4 philosophers sit at once, using asymmetric rules (some pick right fork first), or using a semaphore/monitor to arbitrate.
